Tell me about when you were blocked on a simple problem:

One challenging problem I faced was adding a function expression to an object. Firstly, I didnt understand the idea of adding a function as a property, and secondly I was perplexed by the way that my solution worked.

You can see in the image how athleteArray has 2 elements, with

, however when I run the win() function, the output says
'Chris won the High Jump event'
twice:

My though progression was:

  • At least the output is running twice, so that means both function calls are working.
  • It seems like the values 'Chris' and 'High Jump' are persisting, even though I called the function with 'Santa' and 'delivery'.
  • I tried to extract the variables into new variables to inject into the sting. This also didn't work.
  • After a lot of trial and error, and reading a few articles, I went to ChatGPT:

The explaination from ChatGPT makes sense. Essentially, when the for loop runs, it keeps the final values of athleteArray[i].name and fsport = athleteArray[i].sport, which means that they are used then the win() function is used on both 'Santa' and 'Chris'.

Essentially, the solution calls an outer function, that returns an inner function. The outer function has the values of 'Santa/delivery' and 'Chris/High Jump', so that when the inner function is run, it is using those values at the time, rather than the final values of name and sport from once the for loop is finished running.

I had not come across this topic before, which is called 'closure', which is to use the variable values at each pass of the for loop, rather than waiting for the final values from the loop. This was an interesting and good learning for me.

Reflect on how confident you feel using each of these problem-solving techniques/processes:

Pseudocode

I would say that I dont often use this technique except for reaaally difficult problems. I would probably use it last in the case that I got really stuck. I can however understand the benefits of breaking down a problem into steps and writing those steps down, as that process seems to help the problem solving by itself.

Trying something

This is probably my go-to approach, where I try something quickly to see if it will work. Quite often it does not. lol. I guess on the other hand as I develop my skills, it could be useful to just type in what I think, and I imagine with more experience, the solution will gradually become more likely correct.

Rubber ducky method

I like this method. Similar to Pseudocode, it helps to talk through a problem step by step. I would say that I do not use this tool often, excet for the case when a problem is very very difficult with a lot of difficult steps.

Reading error messages

Error messages are useful. I read them often. I would say they are more useful for finding punctuation mistakes, or missing brackets and colons. Error messages are not as useful for solving complex problems, as the interpreter does not understand what it is that I am trying to achieve, but mainly whether it is gramatically correct, and able to be run by the browser.

Console.logging

I use console.log() often to see what is the value of an object or item at a given step.

Googling

I use googling often to search for solutions and explainations to technial questions.

Asking your peers for help

To be honest, I am a bit of a lone ranger, so I would prefer to solve something myself, than to ask anyone else. I will keep this in mind for future reference.
